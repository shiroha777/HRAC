# HSM-Adaptive 超参数说明

## 一、需要人为调整的主要超参数（4个）

### 1. `rho` - 历史 EMA 衰减系数
- **默认值**: `0.9`
- **作用**: 控制历史信息累积的衰减速度
  - `T_i = rho * T_{i,t-1} + (1-rho) * align_centered[i]`
  - `r_i = rho * r_{i,t-1} + (1-rho) * ||Δ'_i||`
- **取值范围**: `(0, 1)`
- **推荐值**: 
  - `0.9` - 标准设置（保留更多历史）
  - `0.95` - 更保守（更依赖历史，适合稳定场景）
  - `0.8` - 更激进（更快适应新变化）
- **何时调整**: 
  - 攻击者变化频繁 → 降低 `rho`（更快响应）
  - 需要长期跟踪 → 提高 `rho`（更依赖历史）

### 2. `beta` - 服务器动量衰减系数
- **默认值**: `0.9`
- **作用**: 控制服务器动量的衰减速度
  - `server_momentum = beta * momentum_{t-1} + (1-beta) * aggregated`
- **取值范围**: `(0, 1)`
- **推荐值**: `0.9`（标准动量设置）
- **何时调整**: 通常不需要调整，除非训练非常不稳定

### 3. `client_momentum_beta` - 客户端动量衰减系数
- **默认值**: `0.9`
- **作用**: 客户端侧动量 EMA（目前代码中未直接使用，但保留接口）
- **取值范围**: `(0, 1)`
- **推荐值**: `0.9`
- **何时调整**: 如果实现客户端侧动量，可能需要调整

### 4. `tau_quantile` - 参考分位数
- **默认值**: `0.3`
- **作用**: 仅用于日志参考（`tau_t = quantile(trust_scores, tau_quantile)`）
- **重要**: **不影响实际聚合**，只用于日志和监控
- **取值范围**: `(0, 1)`
- **推荐值**: `0.3`
- **何时调整**: 通常不需要调整（仅用于日志）

## 二、硬编码的常数（可能需要调整）

### 5. `sketch_dim` - CountSketch 维度
- **当前值**: `256`（硬编码在代码中）
- **作用**: CountSketch 投影的目标维度
  - 更大的维度 → 更精确，但内存/计算略增
  - 更小的维度 → 更快，但可能丢失一些结构信息
- **推荐值**: `256`（已在代码中固定）
- **何时调整**: 
  - 模型非常大（>1M 参数）→ 可以降到 `128`
  - 需要更高精度 → 可以提高到 `512`
- **注意**: 这个值目前硬编码，需要改代码才能调整

### 6. `r_prev` floor - 历史尺度下限
- **当前值**: `0.1 * median_norm`（第1067行）
- **作用**: 防止历史模长过小导致 log-ratio 爆炸
- **推荐值**: `0.1` 倍中位数（代码中固定）
- **何时调整**: 通常不需要

### 7. `T_mad` floor - 历史 z-score MAD 下限
- **当前值**: `1e-3`（第1086行）
- **作用**: 防止 MAD 过小导致 `T_z` 爆炸
- **推荐值**: `1e-3`（代码中固定）
- **何时调整**: 通常不需要

### 8. 最小权重 (`1e-3`)
- **位置**: 
  - `w_hist`（第1088行）
  - `w_sketch`（第1126行）
  - 最终 `weights`（第1133行）
- **作用**: 保证所有权重非零（软权重设计）
- **推荐值**: `1e-3`
- **何时调整**: 
  - 需要更激进的过滤 → 降低（如 `1e-4`）
  - 需要更保守（避免误伤）→ 提高（如 `1e-2`）

### 9. `w_global` 计算中的倍数 (`5.0`)
- **当前值**: `5.0`（第1137行）
- **作用**: `w_global = tanh(std(alignments) * 5.0)` - 控制对齐度标准差的影响
- **仅用于**: Trust 分数（日志），**不影响实际聚合**
- **推荐值**: `5.0`
- **何时调整**: 通常不需要（仅用于监控）

### 10. Trust 分数中 scale 的权重 (`0.5`)
- **当前值**: `0.5`（第1138行）
- **作用**: `trust = w_global * align_centered - 0.5 * scale_z_scores`
- **仅用于**: Trust 分数（日志），**不影响实际聚合**
- **推荐值**: `0.5`
- **何时调整**: 通常不需要（仅用于监控）

### 11. CountSketch 随机种子 (`12345`)
- **当前值**: `12345`（第1102行）
- **作用**: 固定 CountSketch 的哈希函数（`h`, `s`），确保跨轮一致性
- **推荐值**: 固定值（无需调整）
- **何时调整**: 从不调整（固定种子保证可重现性）

### 12. `eps` - 数值稳定性常数
- **默认值**: `1e-12`（第806行）
- **作用**: 防止除零和数值不稳定
- **推荐值**: `1e-12`
- **何时调整**: 通常不需要

## 四、推荐配置

### 标准配置（适用于大多数场景）
```python
aggregation = C_HSM_FedAvg(
    honest_nodes=honest_nodes,
    byzantine_nodes=byzantine_nodes,
    rho=0.9,              # 历史 EMA 衰减
    beta=0.9,             # 服务器动量衰减
    client_momentum_beta=0.9,  # 客户端动量（目前未使用）
    tau_quantile=0.3,     # 仅用于日志
)
```

### 快速响应配置（攻击者变化频繁）
```python
rho=0.8  # 降低历史权重，更快适应新变化
```

### 长期跟踪配置（需要更强的历史记忆）
```python
rho=0.95  # 提高历史权重，更依赖长期行为
```

## 五、关键点

1. **自适应设计**: 大部分权重和阈值都是自适应计算的（基于数据分布）
2. **最小超参**: 真正需要调整的只有 `rho`（有时可能需要调整）
3. **硬编码常数**: 大部分是数值稳定性常数，通常不需要调整
4. **Trust 分数参数**: `w_global` 和 scale 权重仅用于日志，不影响聚合

---

## 六、参数敏感性分析

### 高敏感参数
- ⚠️ **`rho`**: 影响历史累积速度，可能显著影响长期攻击检测

### 中等敏感参数
- `sketch_dim`: 影响 CountSketch 精度，但对大多数场景影响不大
- 最小权重 (`1e-3`): 影响软权重的下限

### 低敏感参数
- `beta`: 标准动量设置，通常不需要调整
- `tau_quantile`: 仅用于日志，不影响聚合
- 其他数值稳定性常数

---

## 总结

**HSM-Adaptive 的超参设计非常简洁**：
- ✅ **真正需要调整的只有 `rho`**（根据攻击模式）
- ✅ 其他参数都有合理的默认值
- ✅ 大部分权重和阈值都是自适应的（基于数据分布）

这比传统方法需要手动调整多个阈值/权重要简单得多。

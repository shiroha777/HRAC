# 论文 2012.10333（Byzantine-Robust Optimizer）复现配置说明

参考论文：**Learning from History for Byzantine Robust Optimization** (arXiv:2012.10333v3)  
代码仓库：https://github.com/epfml/byzantine-robust-optimizer

## 一、论文 / 官方仓库中的实验配置

### 1.1 CIFAR-10 实验（Section 7.2, Fig. 4）

| 项目 | 论文/仓库配置 |
|------|----------------|
| **数据集** | CIFAR-10 |
| **模型** | **ResNet-20**（`get_resnet20`, 无 GroupNorm 时 `gn=False`） |
| **Worker 数** | **25** |
| **Byzantine 数** | **5**（IPM 攻击时用 **11**） |
| **每 worker batch size** | **32** |
| **Epochs** | **100** |
| **学习率** | **0.1**，第 75 个 epoch 后变为 **0.01**（`MultiStepLR(milestones=[75], gamma=0.1)`） |
| **Momentum β** | 0 / 0.9 / 0.99（传统形式：`m_t = g_t + β * m_{t-1}`） |
| **优化器** | SGD，**无** weight decay（官方 `cifar10-all.py` 里 server 用 `SGD(model.parameters(), lr=LR)`） |
| **数据划分** | IID，`DistributedSampler` 按 rank 均分 |

### 1.2 聚合器（论文/仓库）

- **CC (Centered Clipping)**：`tau=100`, `n_iter=1`
- **TM (Trimmed Mean)**：`b = N_BYZ`（5 或 11）
- **RFA**：内迭代 `T=3`
- **Krum**：`n=25`, `f=N_BYZ`, `m=1`
- **CM**：Coordinate-wise Median

### 1.3 攻击

- **BF**：Bit Flipping（Byzantine 发 `-gradient`）
- **LF**：Label Flipping（`revertible_label_transformer: target → 9 - target`）
- **IPM**：Inner Product Manipulation，`epsilon=0.1`，**11 个 Byzantine**
- **ALIE**：A Little Is Enough，`n=25`, `m=5`

### 1.4 官方复现步骤（README）

1. 把 `codes/` 加入 `PYTHONPATH`
2. `pip install -r requirements.txt`
3. 进入 `momentum-robustness/`
4. 运行 `bash cifar10-all.sh`，选 2/3/4/5（cprfa / tmcm / krum / avg）
5. 运行 `bash cifar10-CC-HP-explore.sh` 选 2（CC 超参探索）
6. 结果在 `outputs/` 对应目录

## 二、你当前 LPA 配置与论文的差异

| 项目 | 论文/仓库 | 当前 LPA |
|------|-----------|----------|
| Worker 数 | 25 | 10 (`node_size=10`) |
| Byzantine 数 | 5（IPM 用 11） | 3 (`byzantine_size=3`) |
| 模型 | ResNet-20 | CNNModel（小 CNN） |
| 训练量 | 100 epochs | 200 rounds（且 `total_iterations = display_interval * rounds`） |
| 学习率 | 0.1 → 75 epoch 后 0.01 | 固定 `1e-2`（无 ladder） |
| Momentum | β=0/0.9/0.99 | `alpha=0.1`（对应 β=0.1，且与论文常用 0.9/0.99 不同） |
| CC 阈值 | tau=100 | IID 时 threshold=0.1，noniid=0.3（量纲/含义与论文 τ 不同） |
| Weight decay | 无 | 有（如 `weight_decay=0.0085`） |

LPA 里 momentum 形式为：`m = (1-alpha)*g + alpha*m`，即 **alpha 对应论文的 β**（alpha=0.9 ⇔ β=0.9）。

## 三、在 LPA 中尽量对齐论文的设置（建议改法）

若要得到与论文更可比的结果，建议在 LPA 里做如下调整。

### 3.1 节点与 Byzantine

- **`main CMomentum.py`**  
  - `node_size = 25`  
  - `byzantine_size = 5`（跑 IPM 时改为 11，并保证攻击里用 11 个 Byzantine）

### 3.2 模型

- 在 **`ByrdLab/tasks/neuralNetwork.py`** 中增加 **ResNet-20**（或从 byzantine-robust-optimizer 的 `codes.tasks.cifar10.get_resnet20` 对齐实现），CIFAR-10 实验时用 ResNet-20 替代当前 CNNModel。

### 3.3 训练轮数与学习率

- **rounds**：对应 100 个 epoch。  
  - 论文：25 workers，每 worker batch 32 → 每 epoch 约 `50000/(25*32)≈62` 个 batch，100 epoch ≈ 6200 步。  
  - LPA 每 iteration 是每 node 一个 batch：10 节点时每 epoch 约 `50000/(10*32)≈156` 步，25 节点时约 62 步。  
- 建议：**`rounds = 100`**，**`display_interval`** 设为 10 或 20（保证 `total_iterations = display_interval * rounds` 与上面步数一致；若按 100 epoch、62 步/epoch 则 `total_iterations = 6200`，即 `display_interval * rounds = 6200`）。
- **学习率**：初始 **0.1**，第 75 个 epoch 后 **0.01**。  
  - 在 **`main CMomentum.py`** 使用 **`ladder_lr`**，例如：
    - `decreasing_iter_ls = [0, 75*62]`（75 epoch 对应的 iteration 数，62 为 25 节点时每 epoch 步数）
    - `proportion_ls = [1.0, 0.1]`  
  - 并在 **`ByrdLab/tasks/neuralNetwork.py`** 里把 **`super_params['lr']`** 设为 **0.1**。

### 3.4 Momentum

- 论文 Fig. 4 中 β=0 / 0.9 / 0.99。  
- 在 **`ByrdLab/tasks/neuralNetwork.py`** 的 **`super_params`** 里把 **`alpha`** 设为 **0.9** 或 **0.99**（对应 β=0.9/0.99）；若要做无 momentum 对比则 **alpha=0**（或等价地不累加历史）。

### 3.5 Centered Clipping (CC)

- 论文/仓库：**tau=100, n_iter=1**。  
- 在 **`main CMomentum.py`** 里选 **`cc`** 聚合时，给 **`C_centered_clipping`** 传 **`threshold=100`**（与论文 τ 一致）；若你当前实现是“相对阈值”，需要确认与论文的 clip 半径定义一致（论文是绝对半径 τ）。

### 3.6 攻击与数据划分

- 攻击名称与论文一致：**bf / ipm / alie / label_flipping**，IPM 用 **epsilon=0.1**，IPM 时 Byzantine 数 11。  
- 数据划分：**IID**（`iidPartition`），与论文一致。

### 3.7 Weight decay

- 论文官方 CIFAR-10 脚本未用 weight decay。为严格复现，可在 LPA 中把 **`weight_decay`** 设为 **0** 做一组对比。

## 四、论文中“如何配置得到和论文一样的结果”的要点

1. **严格按仓库**：用 byzantine-robust-optimizer 的 `cifar10-all.sh` + `cifar10-all.py`，不改 N_WORKERS、N_BYZ、EPOCHS、LR、BATCH_SIZE、ResNet-20、`Clipping(tau=100, n_iter=1)` 等。  
2. **随机种子**：仓库用 `--seed 0`（或 1, 2）做多次 run，复现时固定相同 seed。  
3. **学习率调度**：`MultiStepLR(milestones=[75], gamma=0.1)`，即 75 epoch 后 lr 乘 0.1。  
4. **Momentum**：传统形式 `m_t = g_t + β*m_{t-1}`，β∈{0, 0.9, 0.99}。  
5. **Byzantine 数**：非 IPM 用 5，IPM 用 11。  
6. **数据**：CIFAR-10 IID，每 worker batch=32，drop_last。

在 LPA 中按第三节逐项对齐以上几点，即可在本地得到与论文/仓库更一致的配置；若再对齐 ResNet-20 与无 weight decay，数值上会更容易接近论文曲线。
